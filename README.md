# Birdex

A Pokédex of **real-world birds**, built from your [eBird](https://ebird.org) life list.
Fully client-side, no build step, no server, no account — an installable offline PWA
hosted on GitHub Pages.

**Live:** https://jqntn.github.io/birdex/

## How it works

1. Download your eBird life list as CSV:
   <https://ebird.org/lifelist?r=world&time=life&fmt=csv>
2. Open Birdex and drop the file in. Everything is parsed locally in your browser —
   the CSV never leaves your device.
3. Every species you've seen lights up in the dex. Browse all ~11,000 birds of the
   world, track completion by region, earn badges, and find your shinies.

## Features

- **Full world dex** (~11,167 species from the eBird taxonomy) with dex numbers in
  taxonomic order, French names (English fallback), and fuzzy search.
- **Region scoping** — completion measured against any country / continent / the world.
- **Rarity tiers** derived from real-world range (Common → Legendary), plus a
  deterministic 1-in-512 *shiny* layer on your own sightings.
- **Achievements** — count milestones, family/order medals, geography, rarity, big days.
- **Stats** — completion bars, lifers-per-year, top families, rarity breakdown, countries.
- **Detail view** with a lazily-loaded Wikipedia photo (falls back to a silhouette offline).
- **Installable PWA** — works fully offline after first load.

## Tech

Vanilla JavaScript ES modules, one CSS file with custom-property theming, a service
worker, and [Fuse.js](https://fusejs.io) for search. No framework, no bundler.
Data is vendored as static JSON, generated by the scripts in `tools/`.

## Regenerating data (`tools/`)

The taxonomy is keyless; per-region species lists need a free
[eBird API key](https://ebird.org/api/keygen).

```sh
cd tools
cp .env.example .env        # then put your key in EBIRD_API_KEY=...
node buildAll.js            # taxonomy + regions + rarity + version stamp
node makeIcons.mjs          # (re)generate PWA icons
```

Without a key, `buildAll.js` still produces a working world-only build (rarity marks
extinct species as Legendary; regions populate once you add the key and re-run).
`.env` is gitignored — **the key is never committed or shipped.**
